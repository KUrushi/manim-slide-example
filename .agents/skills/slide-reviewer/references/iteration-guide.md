# 改善フィードバック → 他スキルへの振り分けガイド

## 概要

slide-reviewer のレビュー結果から具体的な改善アクションへマッピングするためのガイド。
各発見カテゴリに対して、どのスキルを使って改善すべきかを明記する。

---

## 発見カテゴリと対応スキル

| 発見カテゴリ | 対応スキル | 具体的アクション |
|-------------|----------|---------------|
| メッセージ不明確 | slides-composer | slides.md のスライド内容を書き直す |
| テキスト過多 | slides-composer | 複数スライドに分割、情報の取捨選択 |
| 視覚階層が弱い | manim-slides-best-practices | font_size、色、配置を調整 |
| アニメーション混乱 | manimce-best-practices | アニメーションシーケンスを簡素化 |
| ペーシング問題 | slides-composer | スライド順序を変更、トランジションスライド追加 |
| コントラスト不足 | manim-slides-best-practices | カラーパレット定数を調整 |
| ナラティブの断絶 | slides-composer | slides.md 全体の構成を見直す |
| 前提知識不足 | slides-composer | 導入スライドを追加 |
| 感情的アークが弱い | slides-composer | ストーリー構成を再設計 |
| 第一印象が弱い | manim-slides-best-practices | タイトルスライドのデザインを改善 |

---

## 詳細マッピング

### slides-composer に戻すケース

構成・内容レベルの問題。スライドの「何を伝えるか」「どの順序で伝えるか」に関わる。

**具体例**:
- 「スライド3と4のメッセージが重複している」→ slides.md でスライドを統合
- 「結論が最後まで出てこない」→ ピラミッド原則に基づき構成を変更
- 「聴衆の前提知識が足りない」→ 導入セクションを追加
- 「ペースが一定で退屈」→ 緩急をつけたスライド配置に変更
- 「テキストが多すぎてスライド5が読めない」→ 2-3スライドに分割

### manim-slides-best-practices に戻すケース

視覚・デザインレベルの問題。スライドの「どう見せるか」に関わる。

**具体例**:
- 「フォントサイズが小さくて読めない」→ font_size 定数を調整
- 「背景と文字のコントラストが不足」→ カラーパレットを調整
- 「視覚的な階層がなく、何が重要か分からない」→ フォントサイズ・ウェイトで階層を作る
- 「タイトルスライドが素っ気ない」→ タイトルスライドのレイアウトを改善
- 「Canvas の使い方が不適切」→ Canvas パターンを見直す

### manimce-best-practices に戻すケース

アニメーション・技術レベルの問題。Manim の使い方に関わる。

**具体例**:
- 「アニメーションが速すぎて何が起きたか分からない」→ run_time を調整
- 「不要なアニメーション効果が多い」→ FadeIn/Write 等のシンプルなアニメーションに変更
- 「3D シーンの視点が不適切」→ カメラアングルを調整

## 改善サイクル

```
slide-reviewer でレビュー
    ↓
問題を分類
    ↓
┌─────────────────────────────────────┐
│  構成・内容の問題                      │
│  → slides-composer で slides.md 修正  │
│  → 再実装 → 再レンダリング → 再レビュー  │
└─────────────────────────────────────┘
┌─────────────────────────────────────┐
│  視覚・デザインの問題                   │
│  → manim-slides-best-practices で修正  │
│  → 再レンダリング → 再レビュー           │
└─────────────────────────────────────┘
┌─────────────────────────────────────┐
│  アニメーション・技術の問題              │
│  → manimce-best-practices で修正      │
│  → 再レンダリング → 再レビュー           │
└─────────────────────────────────────┘
```

### 改善の優先順位

1. **Layer 1 (知覚)**: 読めない → 最優先で修正
2. **Layer 2 (理解)**: 伝わらない → 高優先度で修正
3. **Layer 3 (感情)**: 退屈 → 中優先度で修正
4. **Layer 4 (行動)**: 行動につながらない → 全体が改善された後に対応

下位レイヤーの問題を先に解決すること。読めないスライドの感情的アークを改善しても意味がない。
